Two confirmed defects (from 04-VERIFICATION.md):
- **CR-01 — coverage hole.** Both `verify_coref_eval.py` (lines 356-364) and `verify_entity_bridge.check_demo_critical` (lines 413-454) compute the denominator from what is *present* (ground-truth-covered expected ids; hubs that happen to exist) rather than from the authoritative 9-id set. This plan makes the gate able to fail when the bridge is wrong: denominator = 9 in BOTH gates, an absent/unstamped id is a FAILURE, coref measured through the resolution path (mention → bridge resolution → stamped `entity_id`), uncovered ids surfaced as a coverage FAILURE, and the 9-id set consolidated into one shared module so the two gates can never drift (IN-03).
