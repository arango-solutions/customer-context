<objective>
Build `scripts/build_entity_bridge.py` — the offline Python ETL that creates the canonical entity bridge between both graphs.

Purpose: Phase 4's core deliverable. Links the same real-world entity (Account, Contact, Contract, Product) across the structured graph and the AutoGraph KG by building a `canonical_entities` vertex hub + `same_as` edge collection. Closes the `entity_id` gap on `customer360_Entities` (which carry entity_name/entity_type/embedding but no canonical ID until this script stamps one). Every Phase 5 agent tool that traverses the cross-graph join depends on this bridge existing.

Output:

- `canonical_entities` collection in ArangoDB (vertex, keyed on entity_id)
- `same_as` edge collection in ArangoDB (edges, directed leaf → hub)
- `entity_id` field stamped onto matched `customer360_Entities` documents
