<objective>
Fix three structurally hollow guards in the Phase 4 entity bridge — two in `build_entity_bridge.py` (silent last-write-wins dict assignments that discard resolution conflicts before any DB write) and one in `verify_coref_eval.py` (unreachable overall-accuracy exit code). After this plan, a genuine double-resolution conflict causes a hard build failure rather than silent truncation followed by a guaranteed PASS, and the coref eval's exit code matches its docstring.

Purpose: The project's defined failure mode is "a confident wrong answer passes a check that appears to validate it." The Phase 4 integrity gate is currently an instance of that exact failure mode: `check_no_double_resolution` queries the DB after conflicts have already been discarded in memory, so the gate is structurally guaranteed to pass regardless of data quality. This plan makes the guard actually fire on real conflicts — proving it by injecting synthetic conflicts and asserting non-zero exit.

Output:
- `scripts/build_entity_bridge.py` — `build_alias_dict` and `name_to_id` accumulation raise `ValueError` on genuine surface-form → different-entity-id collisions
- `scripts/verify_coref_eval.py` — docstring, printed label, and exit-code semantics made mutually consistent with D-05 ("demo-critical hard gate; overall is informational")
</objective>
