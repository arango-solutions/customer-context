<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| queryText (free-form question text) → BM25 SEARCH / embedding | Untrusted text; reaches AQL only via the aql tag as a bind value inside TOKENS(...) and as the embedding input — never as a collection/analyzer/field name. |
| tool → ArangoDB + OpenAI | Read-only AQL (LIMIT k*4); embedding call sends only the query text, no secrets. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-07 | Tampering | hybridRetrieve AQL (injection via queryText/accountId) | mitigate | queryText, accountId, k, qVec bound via the `aql` tag; collection/view/analyzer names are string literals; the conditional account filter uses the supported `${cond ? aql\`...\` : aql\`\`}` composition (still bind-safe). |
| T-05-08 | DoS | unbounded vector/BM25 scan | mitigate | Both AQL calls LIMIT ${k*4}; k is Zod-bounded (1..20); fusion + traversal operate on the bounded set. |
| T-05-09 | Integrity | retrieval traced to wrong/no account | mitigate | Sourcing traverses PART_OF to a real customer360_Documents node and filters Document.account_id; the wrong-unit sources collection is forbidden (grep gate, Pitfall 1). |
| T-05-10 | Info Disclosure | OPENAI_API_KEY leakage | mitigate | Key read from env, never serialized into retrievalPath/data; only chunk _ids/content/account_id/citable_url returned. |
