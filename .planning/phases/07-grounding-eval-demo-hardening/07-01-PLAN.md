| Boundary | Description |
|----------|-------------|
| eval/judge → OpenAI | Answer text (untrusted model output) is sent to the judge model as the CLAIM to grade |
| judge → ArangoDB | Judge fetches cited record content by `_id` (read-only DOCUMENT lookup) |
| test layer → runtime | The judge must NEVER cross into the runtime answer path (web/ or index.ts/stream.ts) |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Spoofing (fabricated sourcing) | faithfulness judge self-certifying grounding | mitigate | Judge is test-time only and advisory; runs ONLY on the post-`enforceGrounding` envelope; never imported under web/ or in index.ts/stream.ts. Verified by `grep -rl faithfulness web/` returning empty (Pitfall 1). The deterministic `_id` gate stays the authoritative floor. |
| T-07-02 | Tampering | LLM-judge prompt-injection via answer text inflating the score | mitigate | Strict rubric: "do NOT use outside knowledge, do NOT be generous"; abstain = not-supported; `temperature:0`. Test-only, so no production impact even on a manipulated verdict. |
| T-07-03 | Information Disclosure | judge reads ARANGO_*/OPENAI_API_KEY via the eval's env | accept | Judge runs only in the local/CI eval; no env serialized into any output; reuses the eval's single `loadEnv()` (no second loader, Pitfall 5 / T-06-04). |
