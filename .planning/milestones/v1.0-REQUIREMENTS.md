# Requirements: Customer 360 (Graph-Based Demo)

**Defined:** 2026-06-15
**Core Value:** A free-form question only answerable by joining the structured and unstructured graphs returns a correct, fully-sourced answer — every fact traceable to the record and graph it came from.

## v1 Requirements

Requirements for the demo-ready release. Each maps to a roadmap phase.

> **Lean-demo scope note (2026-06-16 pivot):** v1 is the lean ~5–6 wk demo (6-question arc, 2 accounts, ArangoDB-product domain, AutoGraph-hybrid via the deployed platform, curated-AQL only, citation cards). Two requirements below are **deferred to v2** for the lean demo and are NOT renumbered or removed: **AGENT-04** (validated generated-AQL fallback) and the **React Flow** cross-graph traversal-viz portion of the sourcing/UI display (SRC-02 / UI-02 ship as citation cards in v1; the interactive traversal graph is v2). Also note **AGENT-06** (Q11 ordered timeline) is out of the lean 6-question arc — see NEXT-04 below.

### Architecture & Research

- [x] **ARCH-01**: Architecture options (AutoGraph-hybrid vs. fully-custom graph + agent) are evaluated in a decision matrix scoring traceability, updatability, grounding, operational cost/complexity, build effort, and time-to-demo, with a recommendation made neutrally on the evidence (no assumed winner)
- [x] **ARCH-02**: Research determines the unstructured-graph approach (AutoGraph vs. fully-custom chunk→embed→entity-extraction) with documented evidence on traceability, update behavior, and operational cost/complexity (research-only — no application code)
- [x] **ARCH-03**: The available ArangoDB environment version is verified (capability check) to support the required vector + BM25 retrieval path

### Questions & Data Mapping

- [x] **QMAP-01**: The lean **6-question demo arc** is locked — Q7 (product-ladder adoption + ROI, structured-only anchor), Q2 (renewal risk + WHY), Q12 (usage green / sentiment red — centerpiece), Q9 (champion engagement), Q5 (ArangoGraph/GenAI expansion readiness), Q8 (promise vs. delivery) — each verified to require both graphs (Q7 the one intentional structured-only exception). The Q11 ordered-timeline question is dropped from the lean demo (deferred to v2).
- [x] **QMAP-02**: Each locked question is mapped to the specific source systems and records needed to answer it

### Synthetic Data

- [x] **DATA-01**: 2 distinct, coherent synthetic accounts (one healthy/expanding, one mixed — the Q12 account) with believable multi-year ArangoDB-product-ladder stories (invented companies, NOT real ArangoDB customers)
- [x] **DATA-02**: All sources (Salesforce, Snowflake, DocuSign, Slack, Google Docs, email, PDF) are generated from a canonical event-spine + master entity registry with shared entity IDs
- [x] **DATA-03**: An automated referential-integrity / timeline linter passes before ingest (no orphaned references, plausible chronology, cross-source coherence)

### Graphs

- [x] **GRAPH-01**: The structured graph (Salesforce, Snowflake, DocuSign) is modeled and loaded into ArangoDB
- [x] **GRAPH-02**: The unstructured graph (Slack, Google Docs, email, PDF) is built into ArangoDB via the **deployed AutoGraph platform** (import → build → orchestrate to the Layer-3 KG), with a post-build AQL UPSERT stamping `account_id`/`entity_id` onto the built collections (keyed on `filename`/`citable_url`)
- [x] **GRAPH-03**: Both graphs support idempotent (re-)loading — structured via AQL UPSERT, unstructured via AutoGraph `incremental:true` append plus re-run of the post-build `account_id` UPSERT — so data can be updated without a full rebuild

### Entity Resolution

- [x] **ENT-01**: A canonical entity layer links the same entity (account, person, contract) across both graphs via shared IDs / `same_as` edges
- [x] **ENT-02**: Cross-graph entity links are queryable and traceable, so the bridge is visible in the retrieval path

### Agent

- [x] **AGENT-01**: The custom planner + specialists agent decomposes a question and dispatches structured-graph and unstructured-graph retrieval, then synthesizes a combined answer
- [x] **AGENT-02**: The agent answers each locked canonical question correctly, with the answer visibly drawing on both graphs
- [x] **AGENT-03**: Curated, parameterized AQL tools cover the locked question patterns (the deterministic demo backbone)
- [ ] **AGENT-04**: A validated generated-AQL fallback handles novel questions, running read-only with bind parameters behind an EXPLAIN/allowlist safety gate — *DEFERRED to v2 (lean-demo scope; see NEXT-04)*
- [x] **AGENT-05**: The agent performs cross-graph reconciliation (Q12: usage looks green but sentiment is red) — the signature moment
- [ ] **AGENT-06**: The agent reconstructs an ordered, fully-sourced timeline across both graphs (Q11) — *DEFERRED to v2 (Q11 is outside the lean 6-question arc; see NEXT-04)*
- [x] **AGENT-07**: The agent gracefully refuses or signals uncertainty for out-of-scope or unanswerable questions instead of guessing

### Sourcing & Traceability

- [x] **SRC-01**: Every fact in an answer carries source attribution (graph, collection, record id)
- [x] **SRC-02**: The answer exposes the full retrieval path (graph → records → traversal → AQL) — *v1 ships this as citation cards; the interactive React Flow traversal-graph rendering is DEFERRED to v2 (see NEXT-04)*
- [x] **SRC-03**: A user can click a claim to view the underlying source record/thread (inline provenance)
- [x] **SRC-04**: The planner's reasoning steps are visible in the UI (reasoning transparency)

### UI

- [x] **UI-01**: A Next.js/Vercel dashboard presents a free-form natural-language question box
- [x] **UI-02**: Answers render with inline sourcing display (source citation cards + retrieval-path view) — *v1 = citation cards; the React Flow cross-graph traversal viz is DEFERRED to v2 (see NEXT-04)*
- [x] **UI-03**: Responses stream with no dead air and acceptable live-demo latency

### Grounding & Eval

- [x] **EVAL-01**: An output-validation gate verifies each answer's claims are grounded in the retrieved records, run against the locked question set
- [x] **EVAL-02**: Demo hardening — pre-warm, a backup path, and adversarial-question rehearsal — completed before the live demo

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Real / non-synthetic customer data | Off-limits; companies are 100% invented, NOT real ArangoDB customers |
| Pre-built click-only buttons | Defeats the free-form NL premise (PROJECT.md decision) |
| General-purpose "ask anything" chatbot | Unbounded scope = guaranteed hallucination; scope to canonical questions + refuse out-of-scope |
| KG-level history / time-travel as a built feature | Known AutoGraph gap; bake temporality into synthetic data instead |
| Live write-back / actions (create task, send email) | Adds risk for zero value in a read-only credibility demo |
| Real-time streaming ingestion | Complexity with no demo payoff; data is static synthetic |
| Many accounts / large dataset | Coherence degrades; dilutes the 2 curated stories |
| Multi-tenant / auth / RBAC | Demo plumbing, not the point |
| Reproducing Cadence's reference architecture | Designing our own from first principles |
| Predictive ML churn score (black-box model) | Contradicts the "show your work" thesis; use explainable evidence-based risk |
