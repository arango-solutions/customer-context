# Quick Task 260619-f7c: Fix Vercel monorepo build failure

## Problem

Vercel deploy fails with:

```
npm error No workspaces found:
npm error   --workspace=customer360-agent
Error: Command "npm run build" exited with 1
```

Root cause: `web/package.json`'s `predev`/`prebuild` scripts use
`npm run build -w customer360-agent`. The `-w` workspace flag only resolves from
the directory that *declares* the `workspaces` field — the repo root. Vercel's
project Root Directory is `web/`, so npm runs these scripts from `web/`, which has
no `workspaces` field → "No workspaces found". `agent/dist/` is gitignored, so the
agent **must** be compiled at build time; the prebuild step is mandatory.

## Task

Change both `predev` and `prebuild` in `web/package.json`:

- **From:** `npm run build -w customer360-agent`
- **To:**   `npm --prefix ../agent run build`

`npm --prefix ../agent run build` runs the agent's `tsc` build with `cwd=../agent`
— no workspace lookup needed. `tsc` resolves from the hoisted root `node_modules`.
Works from the `web/` root directory both locally and on Vercel (the sibling
`agent/` dir is present in the build container; `outputFileTracingRoot` already
points at the repo root).

**files:** `web/package.json`
**verify:** `cd web && npm --prefix ../agent run build` emits `agent/dist/index.js`
**done:** both scripts use the path-prefix form; agent dist compiles from `web/`
